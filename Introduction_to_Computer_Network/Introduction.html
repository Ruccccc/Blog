    Encryption

    • transforming lettets without changing its information content.
    • used to achieve the information confindentiality.
    • Enciphering plaintext into ciphertext.
    • Deciphering ciphertext into plaintext.
    Categories
    • Secret-key algorithm
      • Symmetric:same secret-key for both encryption and decryption.
      • DES
      • AES(Advanced Encryption Standard)
        • By NIST
        • AES-128, AES-192, AES-256
    • Public-key algorithm
      • Asymmetric: Public Key for encryption and secret key for decryption.
      • RSA
    Ciphering
    • Transposition Ciphering
      • Rearrange characters.
    • Substitution Ciphering
      • Substitution characters.
    DES (Data Encryption Standard)
    • Divide message into blocks of 64 bits.
    • Permute each block with a 64-bit key
      • 1 bit in each byte is for parity check
      • 16 inentical iterations.
    • Inverse original permutation
    • 3DES (Triple DES)
      • DES is no longer considered adequate nowadays.
      • Do DES three times with a longer key.
    RSA
    • The product of two large prime numbers is used as Public Key.
    • And the two prime numbers will be the secret key.
    • Based on the fact that it's difficult to factor the product of two latge prime numbers.

    Message Authentication

    • a methodology to assure data integrity and to authenticate the data origin.
    • MAC (Message Authentication Code)
    • One-way hash function#(#####Hashfunction)
    Hash function
    • message -> (Hash function) -> hash
    • It's difficult to find the message that produce the hash.
    • Unkeyed hash: no secret key.
      • MD5 (MessageDigest5)
      • SHA-1 (SecureHashAlgorithm)
    • Keyed hash
      • HMAC

    Firewall

    • Packet-filtering firewalls (過濾)
      • Filtering rules are based on packet header-forwarding information.
      • Most used
      • 效能高、安全性低
    • Application-level proxy servers (代理)
      • Forward and filter connections for services
      • 效能低、安全性高
    • Stateful inspection firewalls (檢查)
      • Keep track of all information requests coming frome a network, and scan each incoming communication.
      • 效能低、安全性高
    • Hybrid (混合)

    Classified by ICSA

    Circuit Switching

    • Dedicated communication path between two stations
    • A dedicated channel for the connection on each physical link
    • EX. telephone network
      • 打電話前幾秒的空白時間 -> 找路徑
      • 行動電話空白時間長 -> 因為手機會移動

    operation

    • circuit establishment -> Information transfer -> Circuit disconnect

    Characteristics

    • Establish connection path then transmit data
    • Low channel utilization

    cons

    • Most of time the line is idle for bursty traffic.(非連續的資料傳遞)
    • Devices the are connected must trandmit and receive at the same data rate.

    Packet Switching

    • data are transmitted in packet
    • long message will be broke into serveral packets
    • packet: transmit randomly -> arrived
      • each packet has different route.
      • find "the shortest path"
      • At each node: recived -> stored -> pass to the next node.
    • EX. 郵件

    advantages over Circuit switching

    • Greater efficiency for bursty traffic.
    • data-rate conversion.
    • packet can be all accepted, but delivery delay may increase.
    • Priorities can be used

    Packet

    • Include Header and payload
      • Header : control information (信封)
      • payload : data (內容)
    • Datagram (packet's different name in different layer)
      • Each packet is treated independently
      • packet may arrive out of sequence
      • may be lost or destroyed
        • 重傳 (non-real-time) TCP
        • 忽略 (real-time) UDP
    • packet size
      • 43 bytes : 3 bytes for header, 40 bytes for data.
      • shorter, faster (切太多反而慢)

    Delay

    • Propagation delay
      • typically \(2 \times 10^8\ m/s\)
    • Transmission time
    • Processing delay

    Layered Architecture

    • break task into subtasks (模組化)
    • In a protocol architecture, the subtasks are arranged in a vertical stack.
    • each layer performs a related subset of functions.
    • rely on the next lower layer to perform more primitive function and to conceal the details.
    • change in one layer does not require changes in other layers
    • 每一層都不知道其他層的內容
    • protocol 協定 (同層的溝通規則)
    • OSI 七層
    • TCP/IP 五層
    • 下到上 L1~L5

    OSI v.s. TCP/IP

    OSI model

    • Open System Interconnection
    • By ISO
    • seven layer

    Physical layer

    • transmission a sequence of bits
    • Bits <--> Signal

    Data Link Layer

    • convert unreliable bit from physical layer to a high-level link
    • Error-Free in both directions
      • error detection
      • error correction
    • Retransmission
      • In link layer
      • In transport layer
    MAC(Medium Access Control)

    sublayer

    • 避免資料碰撞
    • Shared media rather than "point to point"
    • Multi-access communication
      • Need link layer to provide error-free packet

    Network layer

    • Using the imformation provide by higher layer to generate header.
    • routing and flow control (based on the header)
      • Flow/congestion control : 路徑選擇/控制流量
        • congestion occurs when users ask more than the network can provide
        • Control the flow of packets into the network
          • spread (分散)
          • buffer (閘道紅綠燈)
    • generate control packets (e.g. ICMP)
    • all nodes must work together in network layer

    Transport layer

    • provide virtual end-to-end link
      • end-to-end flow control
      • retransmission when link or node failure.
    • break and reassemble packet and messages.

    Session layer

    • establishing a session
    • replace by applicaion.

    Presentation layer

    • Encryption, compression, code conversion.
    • replace by applicaion.

    Application layer

    • anything you want
