Ingredients of LAN

Topology

Bus Tree Ring Star

Bus ans Tree

  • MAC protocol: regulate which station on the medium may transmit at any point in time
    • Centralized
    • Distributed
  • Frames will be absorbed by the terminator
  • Multipoint
  • Signal balancing - strength of signal
    • Divide the medium smaller segments
  • Repeater
    • First Layer
    • Usually bi-directional
    • In ring reperater is ued as device attachment, and is undirectional
  • 10BASE 5, 10BASE 2

Ring

  • Repeaters joined by point-to-point links
  • Unidirectional
  • Destination copies the frame into its local buffer
  • The frame circulates until it returns to the source
  • Token ring
    • Won't collide
    • Have to wait for token
    • mangement of token -> token might disappear or stuck somewhere
  • seldom use now

Star

  • Hub: central element, acts as repeater.
  • Bus logically (Layered bus)
  • Hierarchical configuration
    • header hub (HHUB)
    • intremediate hub (IHUB)

Hub

  • First Layer
  • multi-ports repeater
  • Shared medium bus
  • only one station can transmit at a time
  • recognize a malfuncioning station and cut that station out of the network
  • total capacity is still 10 Mbps

LAN switch

  • Second Layer -> can see the inside of frame.
  • Frame is switched to the intended destination directly.
  • Other unused line can be used at the same time.
  • No change is require to attached devices.
  • Each attached device has a dedicated capacity if the switch has sufficient capacity.
  • The switch scales easily.

Type of switch
  • Store-and-forward switch
    • high delay, low error rate
  • Cut-though switch
    • switch start repating as soon as recognizes the destination address.
    • low delay, high error rate.
  • Adaptive switch
    • begin with cut-though mode
    • change to store-and-forward mode if necessary(error rate is too high)

Protocol Reference Model

  • Mostly, transmission medium is sgared in LAN.
  • MAC layer is not found in traditional layers data link control.
  • Two sublayer in OSI layer 2
    • LLC: logical link control
      • Provide one or more service access points(SAPs), whick is used to communicate with upper layer.
    • MAC: medium access control
      • assemble data, addresss and error-detection fields into frame.
      • disassemble frame and perform address recognition and error detection.
      • Govern access to the LAN transmission medium.

LLC

  • similar to other link layers:
    • transmit PDU between two stations, without intermediate node.
  • different with other link layers:
    • support multiaccess
    • relieve some details of link access by the MAC layer.

MAC

  • Polling 徵詢
  • Round robin (Take turns)
  • Reservation
  • Contention
    • low delay, often collide.
    • mostly used. Since internet is bursty.

Two Level Addressing

  • MAC address: identify a physical interface form the station to the LAN.
  • LLC address: indentify an LLC user
    • LLC address(LLC SAP) is associated with a particular user(process, etc.) within a station.

Ethernet LANs

802.3

  • Intended for use in commercial and light industrial environments.
  • Defines physical layer and MAC layer(CSMA/CD)

Pure ALOHA

  • For packet radio networks.
  • After sending a frame, it resends if no ack return in the maximum possible round-trip propagation delay on the network.
  • Collision if two or more simultaneous transmissions.
  • Assume the transmition time of a frame is L. The maximum collide time is 2L.
  • The maximum throughput is only about 18%.

Slotted ALOHA

  • Divide time axis into time slots, transmission is permitted only at the begin of a slot boundary.
  • Assume the length of a time slot is L. The maximum collide time is L.
  • The maximum throughput is about 37%.

CSMA (carrier sense multiple access)

  • Check if the medium is in use or not, if it's in used wait, else transmit.
  • Collision if two or more stations attempt or transmit at about the same time
  • Maximum throughput can far exceed that of Pure/slotted ALOHA

persistent

  • non-persistent
    1. Transmit if the medium is idle.
    2. wait random time if the medim is busy then try again.
    • Capacity is wasted in the time after the end of a transmission before the next try.
    • Less collide, more waste, not FIFO guaranteed.
  • 1-persistent
    1. transmit if the medium is idle.
    2. if the medium is busy, continue to listen until the channel is sensed idle. Then transmit immediately.
    • Collision is guaranteed if more than two stations are waiting.
  • p-persistent
    1. Waiting until the medium is idle. After that, Transmit with probability p, or delay,
    2. After delay, repeat 1.
    • n stations, np is the expected number of stations that will transmit.
    • if np > 1, there will be a collision.
    • Thoughout will drop to zero.
    • np must be less than one for a heavy load
    • Smaller p is, the longer delay will be.

Backoff

  • Binary expontential backoff
    • in the n-th delay, random time from [0, 2^n].
    • the station gives up after 16 unsuccessful.

CD (Collision Detection)

  • If a collision is detected, tramsmit a brief jamming signal to assure all stations know there is a collision and stop transmission.
  • A station detect a collision twice the end-to-end propagation delay

CSMA/CD

  • IEEE 802.3 used 1-persistent.
  • Binary exponential backoff
  • Efficient over a wide range of loads
  • Stations with fewer collisions will transmit before the station waited longer.
  • 後傳的人成功機率高,因為兩次之間等待時間短

Carrier Sense and Collision Detection

  • A collision should produce higher voltage swings than those produced by singal transmitter.
  • IEEE 802.3 restricts the maximum length of cable to 500m
    • Propergation delay will be long
    • The singal will decay and a collision might not be reconigized.
  • For any hub, if there is signal on more than one input, a sollision is assumed. Collision presence(CP) is generated.
